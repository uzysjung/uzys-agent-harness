| Rule | 설명 |
|------|------|
| git-policy | 브랜치 전략, PR 필수, main 직접 커밋 금지 |

| code-style | 불변성, 소형 파일(800줄 이하), 함수 50줄 이하 |
| error-handling | 명시적 에러 처리. 데이터 파이프라인 실패 복구 |
| pyside6 | PySide6 위젯 규칙, 시그널/슬롯 패턴, 스레딩 |
| data-analysis | 데이터 품질 검증, 재현 가능한 분석, 시드 고정 |
